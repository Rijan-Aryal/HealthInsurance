🏗️ Architecture Overview

This is a Clean Architecture-based .NET 9 Web API with the following layers:

  • Domain — Entities, Enums
  • Application — DTOs, Interfaces
  • Infrastructure — EF Core, Repositories, Background Services
  • Presentation — Controllers, Static Files

Base URL: http://localhost:5249/api

🔄 Policy Lifecycle API

POST/api/policies/{id}/activate
Activate a Draft policy → Active. Auto-sets StartDate & EndDate if not provided.
POST/api/policies/{id}/cancel
Cancel a policy → Cancelled
POST/api/policies/{id}/expire
Manually expire an Active policy → Expired
POST/api/policies/{id}/renew
Renew an Expired policy → Creates new Active version
{ "newEndDate": "2026-01-01T00:00:00Z" }

✏️ Policy Endorsements API

POST/api/policies/{id}/endorse
Add an endorsement to an Active policy. Automatically applies changes.
{ "type": 0, "description": "Increased coverage after review", "oldValue": 500000, "newValue": 750000, "effectiveDate": "2024-06-01T00:00:00Z" }
GET/api/policies/{id}/endorsements
Get all endorsements for a policy

📁 Claims API — Full Workflow

Submitted → UnderReview → Approved → Paid
Submitted → Rejected
UnderReview → Rejected

Basic CRUD

GET/api/claims
List all claims with documents and audit logs
GET/api/claims/{id}
Get a specific claim with full details
POST/api/claims
Create a new claim (starts as Submitted)
{ "policyId": "guid-here", "claimAmount": 50000, "description": "Hospitalization claim for surgery" }
PUT/api/claims/{id}
Update claim (only while Submitted)
DELETE/api/claims/{id}
Delete claim (only Submitted or Rejected)

Workflow Endpoints

POST/api/claims/{id}/submit-to-review
Move claim from Submitted → UnderReview
POST/api/claims/{id}/review
Review claim: Approve or Reject. From UnderReview → Approved or Rejected
{ "action": "Approve", "approvedAmount": 45000, "reviewNotes": "Approved with 10% deduction", "assignedAdjuster": "John Smith", "performedBy": "Manager001" }
POST/api/claims/{id}/pay
Pay an Approved claim → Paid
{ "paidBy": "FinanceDept", "notes": "Bank transfer completed" }
POST/api/claims/{id}/assign-adjuster
Assign an adjuster to a claim
{ "adjusterName": "Jane Doe", "adjusterId": "ADJ-123", "notes": "Assigned for detailed investigation" }
POST/api/claims/{id}/flag
Flag a claim for manual/fraud review
"Suspicious activity detected - verify hospital records"

Document Upload

POST/api/claims/{id}/documents
Upload a document (proof, bill, medical report, etc.) — multipart/form-data
Form fields: file (IFormFile), documentType (Proof/Bill/MedicalReport/PoliceReport/Other), description (optional)
GET/api/claims/{id}/documents
List all documents for a claim

Audit Logs

GET/api/claims/{id}/audit-logs
Get complete audit trail for a claim (who changed what and when)

Query

GET/api/claims/by-status/{status}
Filter claims by status: 0=Submitted, 1=UnderReview, 2=Approved, 3=Rejected, 4=Paid
GET/api/claims/policy/{policyId}
Get all claims for a specific policy

🤖 Background Services

PolicyLifecycleBackgroundService

Runs every 24 hours:

  • Auto-Expiration — Policies past their EndDate are automatically marked Expired
  • Renewal Reminders — Reminders sent 7 days before expiry (tracked in RenewalReminders table)
  • Auto-Renewal — Expired policies with IsAutoRenewal=true are renewed automatically

ClaimProcessingBackgroundService

Runs every 6 hours:

  • Auto-Validation — Submitted claims with documents are auto-moved to UnderReview after 1 hour
  • Urgent Review Flagging — Claims UnderReview for >2 days without an adjuster are flagged
  • Fraud Detection — High-value claims (>100,000) are automatically flagged for review
